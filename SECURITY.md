# Política de Seguridad

## Versiones soportadas

| Versión | Soporte |
| --- | --- |
| 0.1.0-beta | Soporte inicial |

## Reportar vulnerabilidades

Para reportes normales usa GitHub Issues. Si el problema puede exponer datos, ejecutar archivos de forma insegura, afectar instaladores o comprometer a usuarios, evita publicarlo como issue público con detalles explotables.

Si existe un canal privado en el perfil del autor, úsalo para enviar el reporte sensible. Si no existe, abre un issue general sin instrucciones de explotación y solicita un canal privado.

## Qué incluir

- Versión de YomiKomi Tana.
- Sistema operativo.
- Tipo de archivo o flujo afectado.
- Impacto potencial.
- Pasos generales para reproducir.
- Logs o capturas, si no contienen material privado ni contenido protegido.

## Alcance

Esta política cubre YomiKomi Tana Desktop, la página oficial, releases e instaladores publicados en este repositorio.

## Límites

No se promete auditoría inmediata, respuesta en un plazo específico ni seguridad absoluta. Las correcciones se priorizarán según impacto, reproducibilidad y disponibilidad del mantenedor.

## Recomendaciones para usuarios

- Descarga solo desde releases oficiales.
- Verifica el hash SHA-256 publicado en cada release.
- Mantén copias de seguridad de tus archivos originales.
- No abras archivos de origen desconocido si no confías en su procedencia.
